## 1. Maven依赖问题

### 1.1 Spring AI BOM版本找不到

**错误信息：**
```
org.springframework.ai:spring-ai-bom:pom:1.0.0-M4 was not found in https://maven.aliyun.com/repository/public
```

**原因：**
- Spring AI 1.0.0-M4是里程碑版本，阿里云镜像未同步
- 阿里云镜像主要同步主流包，Spring AI等较新包可能没有

**解决方案：**
1. 使用较新的里程碑版本 `1.0.0-M6`
2. 添加Spring官方仓库

```xml
<properties>
    <spring-ai.version>1.0.0-M6</spring-ai.version>
</properties>

<repositories>
    <repository>
        <id>spring-milestones</id>
        <url>https://repo.spring.io/milestone</url>
        <snapshots>
            <enabled>true</enabled>
        </snapshots>
    </repository>
</repositories>
```

---

### 1.2 依赖版本缺失

**错误信息：**
```
'dependencies.dependency.version' for org.springframework.ai:spring-ai-pgvector-store-spring-boot-starter:jar is missing
```

**原因：**
- Spring Boot版本与Spring AI版本不兼容
- 版本对应关系：
  - Spring AI 1.0.0-M4 → Spring Boot 3.2.x
  - Spring AI 1.0.0 (正式版) → Spring Boot 3.3.x

**解决方案：**
- 使用 Spring AI 1.0.0-M6 + Spring Boot 3.2.5

## 2. Docker相关问题

### 2.1 PowerShell中Docker命令语法错误

**错误信息：**
```
一元运算符"--"后面缺少表达式
```

**原因：**
- Bash续行符：`\`
- PowerShell续行符：`` ` ``

**解决方案：**

```powershell
# 方式一：写成一行
docker run -d --name rag-postgres -e POSTGRES_PASSWORD=your_password -e POSTGRES_DB=rag_db -p 5432:5432 pgvector/pgvector:pg16

# 方式二：PowerShell续行
docker run -d `
  --name rag-postgres `
  -e POSTGRES_PASSWORD=your_password `
  -e POSTGRES_DB=rag_db `
  -p 5432:5432 `
  pgvector/pgvector:pg16
```

## 3. AI接口配置问题

### 3.1 URL路径重复

**错误信息：**
```
java.io.FileNotFoundException: https://dashscope.aliyuncs.com/compatible-mode/v1/v1/chat/completions
```

**原因：**
- `base-url` 配置了 `https://dashscope.aliyuncs.com/compatible-mode/v1`
- Spring AI会自动追加 `/v1/chat/completions`
- 导致路径变成 `/v1/v1/chat/completions`

**解决方案：**
```yaml
spring.ai:
  openai:
    base-url: https://dashscope.aliyuncs.com/compatible-mode  # 不要包含/v1
```

## 4. 流式输出问题

### 4.1 中文乱码

**现象：**
```
data:娓data:娓呮€濊矾銆佸涔銆佸涔�
data:鏂扮煡璇嗭紝鎴栬€�
```

**原因：**
- 流式响应（SSE）未强制指定UTF-8编码
- 默认编码可能不是UTF-8

**解决方案：**
添加编码过滤器：

```java
@Configuration
public class WebConfig {
    
    @Bean
    public CharacterEncodingFilter characterEncodingFilter() {
        CharacterEncodingFilter filter = new CharacterEncodingFilter();
        filter.setEncoding("UTF-8");
        filter.setForceEncoding(true);
        return filter;
    }
}
```

---

### 4.2 SSE格式的data:前缀

**现象：**
```
data:我都很乐意！
data:✨  
data:有什么我可以帮你的
```

**原因：**
这是SSE（Server-Sent Events）协议的标准格式，每条消息必须以 `data:` 开头。

**说明：**
- `data:` 是协议格式，不是bug
- 前端负责解析并去掉 `data:` 前缀

**前端解析示例：**
```javascript
const eventSource = new EventSource('/test/ai/stream');
eventSource.onmessage = (event) => {
    // event.data 就是去掉 data: 后的内容
    document.getElementById('output').innerHTML += event.data;
};
```
